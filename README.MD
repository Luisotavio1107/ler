## Módulo 2 - Levantamento de Requisitos (Continuação)

- O que é Levantamento de Requisitos 
    - Transformação as necesssidades do negócio em infomações técnicas 
 
- Requisitos Funcionais e Não Funcionais 
    - RF - O que o sistema vai fazer
    - RNF - Como o sistema vai fazer

- Requisitos, Regras, Restrições 
    - Requisitos: A parte Técnica do sistema
    - Regras: As Necessidade do Cliente
    - Restrinção: Limitações ompostos pela, Linguagem, leis, normas, 


- Modelo de Documentação de Requisitos
  |Campo|Conteudo|Exempo|
  |----|------| ------|
  |  ID  |Código| Rf-001|
  | Título | Valor | Tela de Login |
  | Descrinção | O Ator de ser Capaz de | O Cliente conseguira faz Login |
  | Prioridades | Valor da Prioridade | Alta |
  | Regras Vinculadas | Outras Regras Relacionadas | RN-001 - Banco de Dados |


  ### Técnicas de Levantamento de Requisitos
  - Brienfing
  - BrainStorm
  - Questionário 
  - Entrevista
  - Etnografia 
  - WorkShop
  - Prototipagem 

  #### Briefing 
  - Coleta de Dados Inicial 
  - Não Existe um Formato Pré-Definido 
  - Permite Coletas de Diversas Informações do Cliente 
      - Manual de Identidade Visual (Cores, Logo, Slogan, Fontes...)
      - Concorrentes 
      - prazos 
      - orçamentos 
      - Identidades da Empresa (Missão, Visão, Valores)
      - Publico-Alvo 

#### BrainStorm
- Reunião Criativa com Objetivo de Solucionar um ou mais **Problemas**
- Podem ser Estruturados ou não-Estruturados
- Reunião Curta com Foco Bem Definido 
- Ferramentas
      - Miro
      - Mentimentar
      - OneNote

#### Questionário
- Perguntas Direcionadas ao Cliente, com objetivo de Montar o Escopo do Projeto
(Documentação final do Projeto)
- Formas de Fazer o Questionário
    - Reunião com Perguntas Diretas (Qualitativas)
    - Formulário de Coleta de Dados (Quantitativas)

#### Entrevistas
- Instrumento de Coleta de Dados com Usuário do Projeto 
- Entrevistas podem ser formais e Informais 
- Nessa Etapa é necessário um MVP (Produto Mínimo Viável)

#### Etnografia
- Observação da Cultura da Empresa, a Forma como ela atende e se relaciona com seu público interno e externo.

#### Workshop
- Técnica de Levantamento de Requisitos que acontecem na apresentação de uma MVP,
 para um grupo seleto de pessoas (stakeholders) com intuito de absorver a reação do público ao
 produto/projeto
 - Eventos, Feiras e Woekshops de produtos específicos.

 #### Prototipagem
 - Criação de um elemento de design para simulação de um projeto, com o intuito de testar
 funcionalidade e elementos gráficas antes de começar a escreer código
 - Os protótipos podem ser de
      - Baixa fidelidade (refe): rascunho de um desenho pode ser feito a mão 
      - Média fidelidade (rough): desenhos no computador para marcação de espaços e 
      layouts, não precisa de texto e nem de cores
      - Alta fidelidade: protótipo focado nas aplicações de funcionalidades iguais aos do
      projeto que serão desenvolvidos posteriormente em códigos.
